# NNNN-title-in-kebab-case

- **Status:** proposed | active | superseded
- **Date:** YYYY-MM-DD
- **Superseded by:** (ADR id, or "none")

## Decision

One or two sentences. What we will do, stated as a rule, not a discussion.

## Rationale

Why this rule and not the alternative. Short — the incident below carries the
weight, this section just states the inference from it.

## Evidence / incident that triggered it

The concrete thing that happened: a bug, a measured result, a commit, a
transcript. Cite file paths, commit hashes, or dates. No hypotheticals — if
there's no real incident yet, this ADR should stay `proposed`, not `active`.

## Relevant paths / subsystems

Files or directories this decision constrains or is enforced by.

## Review / invalidation condition

What would make this decision wrong, and how we'd notice. Not "revisit
someday" — a concrete trigger (a metric, a recurrence, a design that no
longer fits).

---

Append-only: once an ADR is committed, don't edit its Decision/Rationale
retroactively. If a decision changes, write a new ADR and set this one's
Status to `superseded` + `Superseded by`.
