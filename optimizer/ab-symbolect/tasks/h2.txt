Bug: parseCSV in src/index.js mishandles records whose quoted fields contain a newline. A quoted field may span what looks like multiple lines, and the whole quoted value (including the newline) belongs to one field in one record. For example, parsing 'name,note\n"Alice","line1\nline2"\n"Bob","ok"' must yield exactly two rows, with the first row's note equal to 'line1\nline2'. Currently the record gets split at the newline. Fix src/index.js so quoted newlines are handled correctly, without breaking any existing tests. Run node --test until all tests pass.
